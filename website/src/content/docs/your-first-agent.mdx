This builds the smallest thing that is still a *real* agent: one tool, run through the real loop
against a logged-in provider. It's the complete [`examples/diceAgent.ts`](/efferent/examples/dice-agent/),
piece by piece. Five concepts, in order: a **tool**, a **toolkit**, a **handler layer**, an
**`AgentConfig`**, and **`runAgent`** — composed at the edge with the `Layer`s the loop needs.

<Steps>

1. **Define a tool.** A tool is an `@effect/ai` `Tool.make` with an **object** `success` struct, a
   shared `failure` struct, and `failureMode: "return"` — so a handler failure comes back to the
   model as a tool *result* instead of aborting the turn.

   ```ts
   import { Tool, Toolkit } from "@effect/ai"
   import { Effect, Schema } from "effect"

   const Failure = Schema.Struct({ error: Schema.String, message: Schema.String })

   const Roll = Tool.make("roll_dice", {
     description: "Roll an N-sided die and return the result.",
     parameters: {
       sides: Schema.Number.annotations({ description: "Number of sides (>= 2)." }),
     },
     success: Schema.Struct({ value: Schema.Number, sides: Schema.Number }),
     failure: Failure,
     failureMode: "return",
   })
   ```

2. **Bundle a toolkit.** One tool here; pass more positionally to `Toolkit.make`.

   ```ts
   const toolkit = Toolkit.make(Roll)
   ```

3. **Write the handler layer.** This binds each tool name to an `Effect`. It's the **seam** where
   runtime dependencies (a `cwd`, the `FileSystem` port, …) get injected. Here the handler is pure,
   so it needs none.

   ```ts
   const handlerLayer = toolkit.toLayer(
     toolkit.of({
       roll_dice: ({ sides }) =>
         sides < 2
           ? Effect.fail({ error: "BadInput", message: "need >= 2 sides" })
           : Effect.succeed({ value: 1 + Math.floor(Math.random() * sides), sides }),
     }),
   )
   ```

4. **Assemble the `AgentConfig`.** A `key` (for cache isolation), a versioned `prompt`, and the
   `toolkit`. This object *is* the "define an agent" contract — efferent's own coding agent builds the
   exact same shape.

   ```ts
   import { type AgentConfig } from "@xandreed/sdk-core"

   const config: AgentConfig<typeof toolkit extends Toolkit.Toolkit<infer T> ? T : never> = {
     key: "dice-agent",
     prompt: {
       name: "dice",
       version: "1.0.0",
       text: "You are a dice assistant. Use the roll_dice tool whenever asked to roll.",
     },
     toolkit,
   }
   ```

5. **Run the loop.** `runAgent` loads history, drives turns until the model stops calling tools, and
   persists the result. Provide the handler layer at the edge.

   ```ts
   import { ConversationId, runAgent } from "@xandreed/sdk-core"

   const program = Effect.gen(function* () {
     const cid = yield* Schema.decodeUnknown(ConversationId)(crypto.randomUUID())
     const result = yield* runAgent(config, cid, "Roll a 20-sided die.").pipe(
       Effect.provide(handlerLayer),
     )
     yield* Effect.logInfo(result.finalText)
   })
   ```

6. **Compose the root.** `runAgent` needs a few `Layer`s from `@xandreed/sdk-adapters`: a conversation
   store, the model router, and the auth + settings stores it reads. `UtilityLlm` is optional (it
   powers fast-tier context digests).

   ```ts
   import { FetchHttpClient } from "@effect/platform"
   import { BunContext, BunRuntime } from "@effect/platform-bun"
   import { Layer } from "effect"
   import {
     LocalAuthStoreLive, LocalFileSystemLive, LocalSettingsStoreLive,
     ModelLive, ModelRegistryLive, StoresLive, UtilityLlmLive,
   } from "@xandreed/sdk-adapters"

   const AppLive = Layer.mergeAll(
     StoresLive,                 // ConversationStore (+ ContextTreeStore) — SQLite by default
     ModelLive,                  // the LanguageModel router
     UtilityLlmLive.pipe(        // optional: fast-tier headroom digests
       Layer.provide(ModelRegistryLive),
       Layer.provide(FetchHttpClient.layer),
     ),
   ).pipe(
     Layer.provideMerge(
       Layer.mergeAll(
         LocalAuthStoreLive,     // creds from ~/.efferent/auth.json
         LocalSettingsStoreLive.pipe(Layer.provide(LocalFileSystemLive)),
       ),
     ),
     Layer.provideMerge(BunContext.layer), // platform FileSystem / Path
   )

   BunRuntime.runMain(program.pipe(Effect.provide(AppLive)))
   ```

</Steps>

<Aside type="tip" title="Run it">
Save as `dice.ts` inside a clone of the repo and `bun dice.ts` (you'll need a credential — run
`efferent`, then `:login`). Or just run the bundled copy: `bun examples/diceAgent.ts`.
</Aside>
