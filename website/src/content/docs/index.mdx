A real agent in a handful of lines: define a tool, bundle it into a config, run the loop.

```ts
import { Tool, Toolkit } from "@effect/ai"
import { Effect, Schema } from "effect"
import { runAgent, type AgentConfig } from "@xandreed/sdk-core"

const Roll = Tool.make("roll_dice", {
  description: "Roll an N-sided die.",
  parameters: { sides: Schema.Number },
  success: Schema.Struct({ value: Schema.Number }),
  failure: Schema.Struct({ error: Schema.String }),
  failureMode: "return",
})

const toolkit = Toolkit.make(Roll)
const handlers = toolkit.toLayer(toolkit.of({
  roll_dice: ({ sides }) =>
    Effect.succeed({ value: 1 + Math.floor(Math.random() * sides) }),
}))

const config = {
  key: "dice-agent",
  prompt: { name: "dice", version: "1.0.0", text: "Roll when asked." },
  toolkit,
} satisfies AgentConfig<any>

// runAgent(config, conversationId, "Roll a d20").pipe(Effect.provide(handlers))
```

The same `AgentConfig` shape powers efferent's own coding agent. The whole loop —
tools, the multi-provider router, context compression, sub-agents, approval — lives in
`@xandreed/sdk-core` as Effects you compose. The [dice walkthrough](/efferent/your-first-agent/)
builds this end-to-end; every snippet on this site is backed by a [runnable example](/efferent/examples/).

## Why Effect

efferent is the wedge: the agent loop, the context tree, headroom compression, the
multi-provider router, and approval are all **composable `Layer`s and Effects** in a pure
domain package — typed errors end to end, no `try`/`catch`, provider selection a runtime
concern. Each agent (the bundled coding agent, your own) is a thin driver that composes those
layers at the edge and hands off to Bun. No build step: Bun runs the TypeScript directly.
