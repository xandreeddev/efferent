**`efferent web`** — a browser-served platform for building interactive experiences with natural language (coding is one use case beside web research, data/math breakdowns, lessons). Canvas-first UI: the agent builds full PAGES (`render_ui` — each `id` a tab; same id updates in place, `mode:'append'` streams sections, `active` controls focus) on a full-viewport stage; the transcript and clickable references live in overlay drawers; a floating dock carries the reply bubble, the live activity strip (phase · current tool · elapsed · interrupt), and the command bar, whose hidden `page` field arrives as a `[viewing:<page-id>]` context marker so follow-ups update the page the user is looking at. **Mermaid diagrams & charts** render client-side from source text (`<pre class="ef-mermaid">`, ```mermaid fences) via a vendored, lazily-loaded mermaid 11 (strict security, theme-token colours). Pages **survive restart/--resume** by replaying persisted `render_ui` tool calls. The web agent runs its **own system prompt** (platform identity; "the answer IS a page — never write an .md/.html to disk to show the user something"), with the rewritten page-builder kit doc (hero/columns/features/stats/steps recipes) and an expanded `ef-*` CSS kit. Everything streams as keyed `hx-swap-oob` fragments over a WebSocket (token→cookie auth on loopback); agent HTML stays allowlist-sanitized (SVG banned — diagrams ride as text). New `ui_render` AgentEvent + a 128KB per-call `HtmlTooLarge` guard. **Direct "claude code" mode**: `efferent code --direct` / `:set agentMode direct` strips the swarm leads so the root codes hands-on with the full toolkit. **New bound**: `subAgentMaxChildren` — a per-run child-spawn cap (atomic counter guard in `run_agent`, model-readable `MaxChildrenReached`); web mode defaults it to 2 with depth 1. New `@xandreed/web` package (private): the pure FE layer — `html` tagged-template components, design tokens → CSS custom properties (all three themes), the sanitizer, vendored htmx + mermaid.
